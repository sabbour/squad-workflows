# Routing Rules

## Default Routes

| Signal / Domain | Route to | Reason |
|----------------|----------|--------|
| Architecture, design decisions, code review | Keaton | Lead — owns technical direction |
| Core library code, GitHub API, workflows, CLI | McManus | Backend Dev — owns implementation |
| Tests, quality, edge cases, CI failures | Hockney | Tester — owns test suite |
| Session logs, decisions merge, history | Scribe | Scribe — owns team memory |
| Work queue, backlog, monitoring | Ralph | Work Monitor — owns pipeline |

## Escalation

- If unclear, route to Keaton (Lead makes the call)
- Multi-domain tasks: spawn Keaton + relevant specialist(s)
